# Prosjektrapport

Maks 10 sider. Det handler om å kommunisere til verden hva man har funnet ut. Ingen gidder å lese lange artikler, så skriv effektivt og godt.

- I praksis $\Rightarrow$ noe som ligner på en vitenskapelig artikkel
- Før så har dere skrevet labrapporter
	- Det er en psuedosjanger
		- unødvendig
		- teit
		- sløsing med tid
- I dag: **Hvordan skrive en vitenskapelig artikkel?**
	- Spesielt struktur
	- Det er ekstremt kjempevanskelig 😫
		- Lærer på pH.d
		- Samme format som fordypning og masteroppgave


## Struktur

tegning!

```
\               /    Introduksjon
  \           /
    \       /
      |   |       Metode/Eksperimentelt
      |   |       Resultat + Diskusjon
      |   |       
      |   |  
      |   |
     /     \      Konklusjon
    /       \
```
Figuren viser hvor generelt/spesifikt hver del er

### Introduksjon

- Forklare den "faglige samtalen"
- Falige kontekst

#### Detaljer

- Faglige samtalen
- Tre deler
	- Generelle
		- Bla bla bla
	- Spesifikke 
		- Hva andre folk har gjort som relaterer til hva vi skal gjøre
	- Det vitenskapelige målet/problemstillingen
		- Nesten et sammendrag
		- Hva er det vi har gjort, hva er det vi ønsker å vise
		- f.eks. "vi har gjort akkurat det samme som de der ^, men mye bedre"
- Flesteparten av referansene er i introduksjonen.
	- Ha med minst 5 i labrapporten
- Introduksjonen er det vanskeligste i hele verden, så ikke start med den (🤯⁉)


### Metode/ekspriment

- Hva har dere gjort
- Hvordan har dere gjort det

#### Detaljer

- Viktige eksperimentelle parametere
	- Mikroskop-typer
	- Programvare
	- Tar litt tid å finne riktig detaljnivå ref dette
- Ikke ha med tekst om hvordan SEM virker som en dum dumming
	- Godt eksempel: "We used hyperspy to analyze the (...) data"

### Resultater

- Hva har dere funnet ut

#### Detaljer

- Bare ha med resultater som belyser problemstillingen deres.
- Ikke ha med alt dere tok opp på labben hvis det ikke tilfører noe
- Kritisk gjennomgang av dataene dere bruker
	- Både de som støtter problemstillingen deres
	- og de som ikke gjør det
- Skal være tolkning av dataene, men på litt lavere nivå enn i diskusjonsdelen

### Diskusjon

- Tolke resultatene for å adressere problemstillingen deres
- Bruke info fra resultat-delen
- Dere skal prøve å overbevise leseren om konklusjonen
- Ta med all data som belyser problemstillingen
- Tolkningene deres kan støttes av referanser

### Konklusjon

- Hva betyr deres funn inn i den faglige samtalen

#### Detaljer

- Lett som en fjert
- Oppsummering av arbeidet i kontekst av den faglige samtalen
- Hvordan passer resultatene deres inn i forskningslitteraturen


### Grensegang resultat og diskusjon

- Avhenger av problemstillingen
- Eks: Effekt av $Ga^-$-milling på permalloy
	- Data er som følger
		- Tem: bilde
		- SEM-EDS: bilde med farger
	- Resultat
		- TEM-data 
			- beregne dimensjon av størrelser
		- SEM-EDS
			- $Fe$ og $Ni$ har gått til forskjellige partikler
			- Størrelse på disse partiklene
	- Diskusjon
		- Ta del-resultatene, diskuter bit for bit
		- Hvorfor skjer dette
			- Gallium?
			- Varme?
			- Gjerne trekke fram referanser

## Grenerelt

- Rød tråd er viktig
- Ikke ta med masse ekstra som ikke egentlig handler om det dere skriver om

## Hvordan finne artikler

- Google Scholar babyy 👶
- Søk på ting 🔍
- Finn en artikkel $\rightarrow$ klikk på **cited by** evt se på de de har sitert

## Hvordan lese vitenskapelige artikler

- Tittel
- Figurer
- Abstract/konklusjon
- Resultat/diskusjon/blablabla 😴
- Introduksjon 💀

## Tips

- Starte
	- Eksperiment/metode
	- Figurer
- Siste
	- Introduksjon
	- Konklusjon
	- Abstract
- Skrivingen er veldig iterativ

	- "Wall of figures" masse figurer etter hverandre uten forklarende tekst i mellom
